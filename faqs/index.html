常见问题

Datatables在给我们项目中带来便捷的同时，在使用过程中也会出现不少问题，有些问题是已知的，我会列举在下面。

获得帮助

  • 问：我有一个问题，我怎么得到帮助？
  • 答：目前有Datatables中文网，方便大家查阅，如果英语好点的可以直接查看原网站，不管是原网站还是中文网，都有丰富的 手册 , 参考,例子 , 极客标签以及下面要说到的常见问题。 如果你不能在此页找到你需要的问题，那么你可以在社区提问 或者发布一个在线调试 (怎么调试？)
  • 小技巧：把下面的超链接拖到书签栏，当打开在线调试后，点击这个即可添加Datatables需要的css和js

最常见问题

  • 问：控制台报错，提示api方法不可用
  • 答：1.10.x版本后，有两种初始化的写法， $().dataTable() 返回的是一个jQuery实例， $().DataTable() 返回的是Datatables的api实例， 详细参考 api手册
  • 问：我现在还在使用1.9.x的版本，去哪里查看文档？
  • 答：旧版本的文档可以查阅这个 网址 ，因为考虑到版本升级，新版的Datatables对旧版做了兼容性处理，大部分的 配置选项方法在新版中依然兼容，但还是推荐直接升级到最新版使用。 1.10.x与1.9.x参数名对照表
  • 问：json数据解析错误
  • 答：这个是由于没有按照json的格式返回或者是缺少Datatables指定的参数，详细参考 调试解决方案
  • 问：“Requested unknown parameter” 错误
  • 答：当你在columns里指定了，但在返回的数据里没有的时候，或者列数不匹配不对应，你会得到这个错误，详细参考 调试解决方案
  • 问：我在ie中会提示错误，但是chrome、ff这些浏览器却不会
  • 答：一般可能是缺少逗号，在ie浏览器下会抛出错误，但是chrome这些浏览器会忽略这些错误

一般初始化

  • 问:我觉得Datatables在加载的时候比较慢，有没有让Datatables更快一点
  • 答：有下列几个方法可以提高Datatables的速度，大家根据自己的情况选择合适的办法
    • 如果你是客户端模式，数据在5000条以内，加上 {% include href/orderClasses.dt %}
    • 如果你是ajax加载数据，数据在50000条以内，加上 {% include href/deferRender.dt %}
    • 如果你数据量很多，那么你需要开启服务器模式 （ {% include href/serverSide.dt %}）
  • 注意：如果你使用的是ie6,7这样的，上面的数字可能会少一些。此外如果你使用的是firefox的firebug，你要关掉它，因为他会给性能造成负面影响
  • 问:表格不能正常显示
  • 答：Datatables所需要的table是比较严谨的，table里必须包含thead和tbody表情。再检查其他html是否有效，控制台有没有错误？初次使用Datatables参考 安装手册
  • 问:表格显示的样式和例子的不同
  • 答：检查css样式文件是否引入正确（名称和路径），还有页面上引入的其他css有没有一并引入
  • 问:如何按照自己的喜好摆放Datatables的控件
  • 答：配置 {% include href/dom.dt %} 属性来自有摆放， 示例

网友遇到的问题

  • 问：为什么表头会缩在一起，刷新页面后又正常了
  • 答：在dt初始化的时候，会用类似jquery $().width()这种函数获取table父容器的宽度，如果父容器的style display属性是none的话，获取到的宽度值就会是0，然后dt就会使用自己默认宽度值100px进行替代运算，结果就会变成这样子。 还要检查css样式是否引入
