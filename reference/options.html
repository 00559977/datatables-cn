选项(Option)

DataTables是一个可高度配置化的类库，可以在生成HTML tables过程中的所有方面实现定制。所有特性可以通过打开、关闭或者定制来满足你对表格所有的要求。定制要先定义一个options，然后传入$().DataTable()构造函数，通过定制options的内容来实现定制。 - 例如下面的代码scrollYDT 和 pagingDT 选项用来允许滚动和禁止分页： 

$('#myTable').DataTable( {
    scrollY: 300,
    paging: false
} );

有非常多的配置项可以用来定制你的表格，在下面的文档中都有说明。 如果需要从整体上感知DataTables的初始化选项使用，可以阅读 手册中的配置项。

控制Datatables的延迟渲染，可以提高初始化的速度。

infoDT

控制是否使用jquerui的样式（需要引入jqueryui的css）

lengthChangeDT

是否允许最终用户改变表格每页显示的记录数。

orderingDT

是否允许DataTables排序。

pagingDT

是否允许表格分页。

processingDT

是否显示正在处理的状态。

scrollXDT

水平滚动

scrollYDT

垂直滚动

searchingDT

是否允许搜索（过滤）功能。

serverSideDT

是否允许DataTables使用服务端(server-side)处理模式。

stateSaveDT

保存状态 - 在页面重新加载的时候恢复状态（页码等内容）。

配置(Options)

名称 说明

deferLoadingDT

延迟请求加载服务端的数据，直到表格第二次绘制。

destroyDT

销毁所有符合选择的table，并且用新的options重新创建表格。

displayStartDT

初始化显示的时候从第几条数据开始显示(一开始显示第几页)。

domDT

定义DataTables的组件元素的显示和显示顺序。

lengthMenuDT

定义在每页显示记录数的select中显示的选项。

orderDT

表格在初始化的时候的排序。

orderCellsTopDT

控制表头的哪一个cell可以应用于该列的排序响应。

orderClassesDT

高亮显示在table的body中排序的列。

orderFixedDT

在表格中总是起作用的排序。

orderMultiDT

是否允许多列排序。

pageLengthDT

改变初始的页面长度(每页显示的记录数)。

pagingTypeDT

分页按钮的显示方式。

rendererDT

显示组件的渲染方式。

retrieveDT

是否允许重新初始化一个DataTables实例

scrollCollapseDT

当显示更少的记录时，是否允许表格减少高度。

search.caseInsensitiveDT

在搜索或者过滤时，是否大小写敏感。

search.regexDT

允许或者禁止对在搜索字符串中出现的正则表达式字符强制编码。

search.searchDT

初始化时的表格的过滤（搜索）条件。

search.smartDT

允许或者禁止DataTables的只能过滤（搜索）功能。

searchDT

设置一个全局的过滤条件。

searchColsDT

给每一列定义一个初始的搜索条件。

searchDelayDT

设定搜索的间隔时间。

stateDurationDT

间隔多长时间保存一次状态。

stripeClassesDT

设置斑马条（奇偶行）的css class.

tabIndexDT

用键盘上的Tab键浏览时的顺序。
